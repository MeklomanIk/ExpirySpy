# GIT

## Определение Git

Git — это распределённая система управления версиями, предназначенная для отслеживания изменений в файлах и координации работы над этими файлами среди нескольких человек. Git позволяет разработчикам эффективно сотрудничать, сохраняя историю изменений и обеспечивая контроль версий.

## Функции Git

- **Распределённость**: Каждый разработчик имеет полную копию репозитория, включая всю историю изменений.
- **Отслеживание изменений**: Git сохраняет историю изменений файлов, позволяя вернуться к предыдущим версиям.
- **Ветвление и слияние**: Git позволяет создавать ветки для разработки новых функций или исправлений, которые можно позже объединить с основной веткой.
- **Совместная работа**: Git облегчает совместную работу над проектами, позволяя нескольким разработчикам вносить изменения одновременно.

## Определения

### Репозиторий

- хранилище для проекта, содержащее все файлы и историю изменений.
- может быть локальным (на вашем компьютере) или удалённым (на сервере, например, GitHub, GitLab).

### Изменение
- разница последнего сохранённого состояния файлов и текущего состояния.
- модификация файлов в репозитории, которая может быть добавлена в индекс для последующего коммита.

### Коммит

- зафиксированное состояние изменений в репозитории Git. Коммит содержит снимок текущего состояния файлов, а также метаданные, такие как автор, дата и сообщение коммита.

### Ветка

- последовательность связанных коммитов
- отдельная линия разработки в репозитории Git, позволяющая работать над новыми функциями или исправлениями, не затрагивая основную (главную) ветку.

## Команды

| Команда Git                 | Описание                                                                                                           |
| --------------------------- | ------------------------------------------------------------------------------------------------------------------ |
| `git init`                  | Инициализирует новый репозиторий Git в текущей директории.                                                         |
| `git clone <url>`           | Клонирует удалённый репозиторий по указанному URL на локальный компьютер.                                          |
| `git add <файл>`            | Добавляет указанный файл в индекс для последующего коммита.                                                        |
| `git commit -m "сообщение"` | Создаёт новый коммит с изменениями, добавленными в индекс, и прикрепляет к нему сообщение.                         |
| `git push`                  | Отправляет локальные коммиты в удалённый репозиторий.                                                              |
| `git status`                | Показывает текущее состояние рабочего каталога и индекса, включая изменённые, добавленные и неотслеживаемые файлы. |
| `git diff`                  | Показывает различия между рабочим каталогом и индексом или между двумя коммитами.                                  |
| `git pull`                  | Получает изменения из удалённого репозитория и сливает их с текущей веткой.                                        |
| `git fetch`                 | Загружает изменения из удалённого репозитория без слияния их с текущей веткой.                                     |
| `git checkout <ветка>`      | Переключается на указанную ветку или создаёт новую ветку, если она не существует.                                  |
| `git branch`                | Показывает список всех веток в репозитории.                                                                        |
| `git merge <ветка>`         | Сливает указанную ветку с текущей веткой.                                                                          |
| `git log`                   | Показывает историю коммитов текущей ветки.                                                                         |
| `git reset <коммит>`        | Откатывает текущую ветку к указанному коммиту, изменяя состояние рабочего каталога и индекса.                      |
