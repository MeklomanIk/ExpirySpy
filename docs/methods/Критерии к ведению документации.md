# Критерии
1. Каждый элемент документации должен быть применим для решения элементарной задачи.
2. Документация должна быть написана на понятном языке, избегая двусмысленностей. Быть конкретной.
3. Документация должна быть структурирована с использованием заголовков, списков и таблиц для улучшения читаемости.
4. Документация должна быть актуальной и регулярно обновляться по мере изменения требований или процессов.
5. Первый уровень заголовков (H1) должен использоваться только для названия документа.
6. Второй уровень заголовков (H2) должен использоваться для основных разделов
7. Третий и последующие уровни заголовков (H3, H4 и т.д.) должны использоваться для конкретных элементов документации.
8. Элементы могут быть вложены друг в друга для создания иерархии информации.
