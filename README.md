## Описание проекта

Данное приложение предназначено для отслеживания статуса таких вещей как еда, одежда, расходные материалы.
Характеризующее свойство таких вещей - они могут испортиться, закончиться со временем или стать небезопасными для использования.

## Проблематика проекта

Не задумываясь о сроке годности тех или иных вещей, человек не только захламляет своё место проживания и подвергает себя риску использования вещей, несущих угрозу здоровью, но и тратит деньги и время на повторную покупку испорченных или израсходованных вещей, а также устранения последствий их использования.

## Требования

### Приложение

#### Пользовательские требования

- Интерфейс пользователя должен быть текстовым с использованием стандартных средств вывода в консоль.

Приложение позволяет пользователю:

- Добавлять новые вещи по категориям с указанием срока годности.
- Просматривать список всех вещей с возможностью сортировки по категориям и срокам годности.
- Удалять, изменять количество или информацию о вещах.

#### Технические требования

- Приложение должно быть разработано на языке C# с использованием .NET Framework.
- Данные о вещах должны храниться в локальной базе данных SQLite.
- Архитектура приложения должна быть модульной, с четким разделением между слоями данных, бизнес-логики и пользовательского интерфейса.
- Код-стайл должен соответствовать рекомендациям Microsoft по стилю кода для C#.

### Документация

- Процесс создания документации должен быть записан в виде заметок или базы знаний.
- В документации для удобства разработки рекомендуется записывать применение инструментов, таких как
  - сочетания клавиш в IDE;
  - команды Git и решаемые ими задачи;
  - особенности C# и .NET, которые были полезны в процессе разработки
  - архитектурные решения и паттерны проектирования, использованные в приложении.
  - используемые источники информации (книги, статьи, сайты).

### Отчёты

Отчёты должны:

- быть сгенерированы в формате PDF с использованием Quarto;
- содержать блок-схемы и диаграммы классов Mermaid и Graphviz;
- соответствовать ГОСТ Р 7.0.11-2011;
- включать титульный лист, оглавление, введение, основную часть, заключение и список использованных источников, в соответствии с примером на сайте ГУАП;

## Установка и запуск

1. Склонируйте репозиторий на свой локальный компьютер.
   ```bash
   git clone https://github.com/meklomanik/ExpirySpy.git
   ```
2. Откройте проект в вашей IDE (например, Visual Studio или Rider).
3. Запустите проект
   ```bash
   dotnet run --project ExpirySpy
   ```
